Introduction to Ontologies

An ontology is a formal representation of knowledge as a set of concepts within a domain, and the relationships between those concepts. In the context of the Semantic Web, ontologies provide a way to describe resources and their relationships in a machine-readable format.

RDF (Resource Description Framework) and RDFS (RDF Schema) are fundamental technologies for building ontologies and representing knowledge on the Semantic Web.

Why Ontologies Matter

Ontologies enable:

  • Knowledge sharing across applications, enterprises, and community boundaries
  • Automated reasoning and inference over data
  • Semantic interoperability between different systems
  • Rich knowledge representation beyond basic database models

RDF Basics

The Resource Description Framework (RDF) is a standard model for data interchange on the Web. RDF has features that facilitate data merging even if the underlying schemas differ.

RDF Triple Structure

RDF represents information as triples:

  • Subject: The resource being described
  • Predicate: The property or relationship
  • Object: The value or related resource
# Example of RDF triples in Turtle syntax
@prefix ex:  .
@prefix foaf:  .

ex:Person1 foaf:name "John Doe" .
ex:Person1 foaf:knows ex:Person2 .
ex:Person2 foaf:name "Jane Smith" .

URI Identifiers

RDF uses URIs (Uniform Resource Identifiers) to name the subjects, predicates, and sometimes objects in statements. This ensures global uniqueness and allows for distributed knowledge representation.

RDF Serialization Formats

RDF can be expressed in multiple formats:

  • RDF/XML
  • Turtle
  • N-Triples
  • JSON-LD

RDF Schema (RDFS)

RDF Schema extends RDF with a vocabulary for describing properties and classes of RDF resources. It provides mechanisms for describing groups of related resources and the relationships between these resources.

Key RDFS Constructs

  • rdfs:Class - Defines a resource as a class for other resources
  • rdfs:subClassOf - Indicates that one class is a subclass of another
  • rdfs:Property - The class of RDF properties
  • rdfs:subPropertyOf - Indicates that one property is a specialization of another
  • rdfs:domain - Specifies the class of the subject in a triple using this property
  • rdfs:range - Specifies the class or datatype of the object in a triple using this property
# Example of RDFS in Turtle syntax
@prefix rdfs:  .
@prefix ex:  .

# Class definitions
ex:Person rdf:type rdfs:Class .
ex:Student rdf:type rdfs:Class .
ex:Student rdfs:subClassOf ex:Person .

# Property definitions
ex:name rdf:type rdf:Property .
ex:name rdfs:domain ex:Person .
ex:name rdfs:range xsd:string .

ex:enrolledIn rdf:type rdf:Property .
ex:enrolledIn rdfs:domain ex:Student .
ex:enrolledIn rdfs:range ex:Course .

Building a Simple Ontology

Let's walk through the process of building a simple ontology for a university domain:

Step 1: Identify Key Concepts

First, identify the main classes in your domain:

  • Person
  • Student (subclass of Person)
  • Professor (subclass of Person)
  • Course
  • Department

Step 2: Define Properties

Identify relationships between classes:

  • teaches (domain: Professor, range: Course)
  • enrolledIn (domain: Student, range: Course)
  • belongsTo (domain: Professor, range: Department)
  • offers (domain: Department, range: Course)

Step 3: Create the RDFS Representation

@prefix rdf:  .
@prefix rdfs:  .
@prefix univ:  .
@prefix xsd:  .

# Class definitions
univ:Person rdf:type rdfs:Class .
univ:Student rdf:type rdfs:Class ;
             rdfs:subClassOf univ:Person .
univ:Professor rdf:type rdfs:Class ;
               rdfs:subClassOf univ:Person .
univ:Course rdf:type rdfs:Class .
univ:Department rdf:type rdfs:Class .

# Property definitions
univ:name rdf:type rdf:Property ;
          rdfs:domain univ:Person ;
          rdfs:range xsd:string .

univ:studentID rdf:type rdf:Property ;
               rdfs:domain univ:Student ;
               rdfs:range xsd:string .

univ:teaches rdf:type rdf:Property ;
             rdfs:domain univ:Professor ;
             rdfs:range univ:Course .

univ:enrolledIn rdf:type rdf:Property ;
                rdfs:domain univ:Student ;
                rdfs:range univ:Course .

univ:belongsTo rdf:type rdf:Property ;
               rdfs:domain univ:Professor ;
               rdfs:range univ:Department .

univ:offers rdf:type rdf:Property ;
            rdfs:domain univ:Department ;
            rdfs:range univ:Course .

# Some instance data
univ:CS101 rdf:type univ:Course ;
           univ:title "Introduction to Computer Science" .

univ:ComputerScience rdf:type univ:Department ;
                      univ:offers univ:CS101 .

univ:ProfessorSmith rdf:type univ:Professor ;
                     univ:name "Jane Smith" ;
                     univ:teaches univ:CS101 ;
                     univ:belongsTo univ:ComputerScience .

univ:StudentDoe rdf:type univ:Student ;
                 univ:name "John Doe" ;
                 univ:studentID "S12345" ;
                 univ:enrolledIn univ:CS101 .
